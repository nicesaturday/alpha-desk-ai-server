# Command: /review-apply

## 목적

현재 브랜치의 PR에 달린 리뷰를 읽고,
리뷰 지적사항을 코드에 반영한 뒤 다시 푸시한다.

`gh` CLI를 사용하여 리뷰를 가져오고,
Claude가 코드를 수정하고, 커밋 후 푸시까지 자동으로 수행한다.

---

## 사용 방법

```
/review-apply
```

PR 번호를 직접 지정할 수도 있다.

```
/review-apply 13
```

---

## 동작 규칙

### 1단계: PR 및 리뷰 조회

PR 번호가 주어지지 않으면 현재 브랜치의 PR을 자동으로 찾는다.

```
gh pr list --head <현재 브랜치> --json number
```

그 후 리뷰 코멘트를 가져온다.

```
gh pr view <번호> --json reviews,comments
gh api repos/{owner}/{repo}/pulls/{번호}/comments
```

### 2단계: 리뷰 분석

가져온 리뷰에서 다음을 파악한다.

- 수정 요청 사항 (Request Changes)
- 일반 코멘트 중 수정이 필요한 항목
- 파일별 인라인 코멘트

각 지적사항을 목록으로 정리하여 사용자에게 보여준다.

### 3단계: 코드 수정

사용자 확인 후, 각 지적사항에 대해 코드를 수정한다.

- 해당 파일을 읽고 수정한다
- CLAUDE.md 아키텍처 규칙을 준수하며 수정한다
- 수정할 수 없는 항목은 사유와 함께 사용자에게 알린다

### 4단계: 커밋 및 푸시

수정이 완료되면:

1. 변경된 파일을 스테이징한다
2. 커밋 메시지를 작성한다
   - 형식: `fix: apply PR #{번호} review feedback`
   - body에 수정한 항목 요약 포함
3. 사용자에게 커밋 내용을 보여주고 확인을 받는다
4. 확인 후 `git push`를 실행한다

---

## 커밋 메시지 형식

```
fix: apply PR #<번호> review feedback

- <수정사항 1>
- <수정사항 2>
- ...

Co-Authored-By: Claude Opus 4.6 <noreply@anthropic.com>
```

---

## 주의사항

- 리뷰 내용을 분석한 뒤, 수정 전에 반드시 사용자에게 수정 계획을 보여주고 확인을 받는다
- 푸시 전에 반드시 사용자에게 확인을 받는다
- 리뷰에 동의하지 않는 부분이 있으면 사용자에게 의견을 물어본다
- 리뷰가 없는 경우 "리뷰가 아직 없습니다"를 알려준다
